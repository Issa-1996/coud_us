  SLIDE 3 — PRÉSENTATION GÉNÉRALE
════════════════════════════════════════════════════════

  TITRE : Qu'est-ce qu'USCOUD Sécurité ?

  Définition :
    Application web dédiée à la gestion des incidents de sécurité
    sur les campus universitaires du COUD à Dakar.

  Objectifs :
    ▸ Centraliser et tracer tous les incidents sur les campus
    ▸ Produire des procès-verbaux officiels (Faux, Constat, Dénonciation)
    ▸ Faciliter le suivi des dossiers par les agents de sécurité
    ▸ Assurer la traçabilité de toutes les actions (audit trail)

  Périmètre :
    ▸ Tous les campus COUD (Dakar, Ziguinchor, etc.)
    ▸ Personnel de sécurité USCOUD
    ▸ Intégration avec la base étudiants UCAD

  Chiffres clés :
    ▸ 3 types de PV gérés
    ▸ 4 rôles utilisateurs
    ▸ 100% des actions tracées et auditées

  SLIDE 5 — AUTHENTIFICATION
════════════════════════════════════════════════════════

  TITRE : Authentification & Gestion des Sessions

  Processus de connexion :

    [1] L'utilisateur saisit son email et son mot de passe
        → Page : login.php

    [2] Vérification en base de données
        → Email unique dans uscoud_pv_utilisateurs
        → Mot de passe vérifié par password_verify() (bcrypt)

    [3] Vérification du statut du compte
        → Compte actif/inactif

    [4] Si première connexion (doit_changer_mdp = OUI)
        → Redirection obligatoire vers changement de mot de passe
        → Mot de passe par défaut "COUD" à remplacer

    [5] Session créée avec :
        → Identifiant utilisateur
        → Nom & Prénoms
        → Email
        → Rôle (admin / superviseur / agent / opérateur)

  Règles de sécurité :
    ▸ Mots de passe hachés (bcrypt - standard PHP)
    ▸ Nouvelle connexion : mot de passe obligatoirement changé
    ▸ Minimum 6 caractères pour le mot de passe
    ▸ Impossible de conserver "COUD" comme mot de passe

  SLIDE 15 — JOURNAL D'AUDIT
════════════════════════════════════════════════════════

  TITRE : Traçabilité & Journal d'Audit

  Principe :
    Chaque action effectuée dans le système est
    automatiquement enregistrée dans un journal d'audit.

  Ce qui est enregistré :
  ┌────────────────────────────────────────────────────┐
  │  POUR CHAQUE OPÉRATION :                           │
  │    • Qui ?      → Identifiant de l'utilisateur     │
  │    • Quoi ?     → Type d'action (Créer/Modifier/   │
  │                              Supprimer/Consulter)  │
  │    • Sur quoi ? → Table et enregistrement concerné │
  │    • Quand ?    → Date et heure précises           │
  │    • D'où ?     → Adresse IP de connexion          │
  │    • Comment ?  → Navigateur utilisé (User-Agent)  │
  │    • Avant ?    → Valeurs avant modification (JSON)│
  │    • Après ?    → Valeurs après modification (JSON)│
  └────────────────────────────────────────────────────┘

  Avantages :
    ✅ Responsabilité claire de chaque action
    ✅ Impossible de nier une action effectuée
    ✅ Aide à la résolution des litiges
    ✅ Conforme aux exigences de gouvernance institutionnelle
    ✅ Permet la reconstitution de l'historique complet d'un PV

  Aucune action n'échappe au journal d'audit.

════════════════════════════════════════════════════════
  SLIDE 17 — SÉCURITÉ DU SYSTÈME
════════════════════════════════════════════════════════

  TITRE : Mesures de Sécurité Implémentées

  ✅ PROTECTIONS EN PLACE

    🔐 Authentification
       • Mots de passe chiffrés (bcrypt — standard industriel)
       • Changement de mot de passe obligatoire à la première connexion
       • Comptes désactivables instantanément

    🛡 Protection des données
       • Requêtes SQL préparées (protection injection SQL)
       • Échappement des données affichées (protection XSS)
       • Dossier de configuration protégé par .htaccess

    👁 Traçabilité
       • Journal d'audit complet (IP, user-agent, avant/après)
       • Horodatage de toutes les actions

    🔑 Contrôle d'accès
       • Restrictions appliquées côté serveur (non contournables)
       • Vérification du rôle à chaque requête
       • Session vérifiée sur chaque page
